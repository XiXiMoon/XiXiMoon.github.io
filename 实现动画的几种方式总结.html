1、Gif图片

特点：直接放上去，没法控制gif图，有闪烁现象

2、序列帧

本质是GIF图单个帧的铺开，结合css animation一种实现方式。

缺点：序列帧在 H5 开发中手机分辨率的不同，可能有轻微抖动现象，可通过将该dom嵌套到SVG 中解决

原因：序列帧 background-position在做位移的时候h5像素转换可能有小数，小数会导致像素点遗漏，就出现了抖动，svg固定了viewbox大小，自然就不会抖动了

7、lottie动画：UI可以生成，是专门用于解析从AE(Adobe After Effects)中通过Bodymovin插件导出的JSON文件，直接渲染动画。

animationData: 通过export 导出的js对象，包含具体的动画实现数据。AE导出的JSON文件
path:动画json对象，使用path便不能使用animationData
loop: true / false / number，循环播放或指定次数
autoplay: true / false，数据加载完成便立即播放
name, 动画名称
renderer: 'svg' / 'canvas' / 'html' 渲染方式
container: 动画所在的dom容器

除了直接引用lottie.min.js库，还可以
import lottie from 'lottie-web';
import animationData from "./loading.json"; //引入你的json动画数据

loadAnimation方法返回的对象：
this.animation.play(); //播放动画
this.animation.pause(); //暂停动画
this.animation.stop(); //停止动画
this.animation.setSpeed(number); //设置播放速度
this.animation.destroy(); //销毁动画

8、WebGL与Canvas

当页面动画复杂性较高时，使用dom进行绘制可能会出现性能问题，画面会出现卡顿，此时可以考虑WebGL或Canvas进行渲染。
